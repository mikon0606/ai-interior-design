# 创始人长期背景信息

## 为什么做这个项目

这个项目最开始来源于一位室内设计师朋友。

设计师和客户沟通方案时，经常需要等待几天才能拿到效果图。

我发现：

- 客户很多时候并不需要最终施工图。
- 他们只是想快速看到一个效果。
- 如果能在短时间内生成效果图，设计师就能更快推进沟通和签单。

因此创建这个项目。

## 项目真正目标

这个项目不是做通用 AI 生图网站。

也不是和 ChatGPT、Midjourney 竞争。

这个项目要做的是：

**AI 室内设计师助手。**

帮助室内设计师：

- 快速出概念图
- 快速出客户提案图
- 快速沟通方案
- 提高谈单效率

## 当前业务模式

用户上传：

- 房间照片
- 户型图
- 设计需求

系统生成任务。

后台查看任务。

人工使用 ChatGPT 等工具生成效果图。

后台上传结果。

用户查看结果。

目前不接自动 AI API。

## 为什么暂时不用自动 API

目前最重要的是验证市场。

不是技术。

如果还没有真实用户，就不应该过早投入自动化和付费 API 成本。

## 当前最重要目标

找到真实设计师用户。

验证：

1. 是否愿意使用
2. 是否愿意持续使用
3. 是否愿意付费
4. 是否能帮助提高谈单效率

## 当前开发原则

优先做：

- 国内访问优化
- 设计师版首页
- 案例展示
- Admin 后台优化
- 提示词模板
- 工作流效率提升

暂缓做：

- 支付系统
- 会员系统
- 自动生图
- 积分系统
- 营销裂变

## 关于技术

技术只是手段。

所有开发决策都应该围绕：

**是否帮助设计师更快获得客户提案图。**

如果一个功能不能帮助设计师提升效率，就降低优先级。

## 未来方向

如果验证成功：

第一阶段：人工审核 + 人工回传

第二阶段：半自动化

第三阶段：自动化生图

但必须先验证真实需求，再扩大投入。
